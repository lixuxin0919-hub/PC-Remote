# PC遥控器 v2.1.1

一款 Android 原生 Bluetooth HID 控制应用，目标是只用手机蓝牙便捷控制 Windows 当前活动窗口，不依赖 Wi-Fi、局域网或电脑端服务。手机会注册为键盘、相对鼠标/滚轮和 Consumer Control 组合设备。

应用采用可配置控制平台架构：既可以控制短视频和音乐播放，也可以作为触控板、快捷键面板和自定义手势遥控器使用。界面采用浅色玻璃质感设计，并针对小屏、横屏、系统语言和滚动性能进行了持续优化。

v2.0.0 已从旧版 Java View 页面重构为 Kotlin + Jetpack Compose，并升级为配置驱动的控制平台。v2.1.1 进一步完成全局滚动性能专项：缩小根状态观察边界、将配置落盘移出主线程，并降低共享玻璃组件的绘制成本。应用名称、界面和文档统一使用“PC遥控器”。

## v2.1.1 滚动性能专项

- 根组合不再直接订阅完整 `AppConfig`；控制模式、全局设置、模式列表、启动模式、架构版本和触感偏好均通过去重的细粒度状态流提供，避免无关配置更新扩大重组范围。
- Activity 不再因蓝牙状态变化重建整套 Compose `setContent` 参数；蓝牙状态改为在应用组合内生命周期感知收集，保持宿主回调对象稳定。
- 配置修改仍立即更新 UI，但 JSON 编码和 SharedPreferences 写入通过单线程 I/O 队列按顺序执行；首次初始化仍同步持久化，避免启动配置丢失。
- 共享卡片、按钮与导航栏降低阴影成本；Slider 轨道渐变缓存到绘制缓存；控制页的控制项过滤、排序与分组按模式版本缓存；自定义图标按 512px 目标尺寸采样并保留内存缓存。
- API 35 / 60Hz Pixel 9 Pro 模拟器、相同 APK 数据和每轮 50 次往返滑动的 5 轮测量中：自定义页卡顿由 30/3907（0.77%）降至 21/3835（0.55%），设置页由 87/3725（2.34%）降至 22/3816（0.58%）。详细方法与限制见 `PERFORMANCE_AUDIT_v2.1.1.md`。

## v2.1.0 更新

- 应用内语言默认“跟随系统”，也可固定为简体中文或 English。Android 13+ 使用系统应用语言 API，Android 9–12 在 Activity 创建前应用本地语言；该偏好不会被控制模式 JSON 导入覆盖。
- 语言切换前先取消触控输入并释放 HID 按键，再安全重建界面。导航、页面、内置模式、默认控件、动作摘要、蓝牙状态、对话框与 Snackbar 均覆盖英文；用户自定义的英文名称保持原文。
- 自绘滑杆只允许从滑块及其触控热区开始水平拖动。点击轨道、轻点滑块或优先发生的纵向滚动不会修改数值；辅助功能 `setProgress` 仍可使用。
- 触控板压力读数只重组右上角压力指示器，不再让整个触控板页面随每个 MotionEvent 重组；英文翻译结果使用 1024 项有界缓存。
- Aurora 背景合并为单一缓存绘制节点，并降低玻璃卡片、按钮和底栏的阴影成本。
- 修复图标位图内部留白导致的视觉过小、按钮禁用/普通状态横向亮带、长英文按钮裁切和 Snackbar 对比度不足。

## 系统要求

- Android 9（API 28）或更高版本
- 手机厂商系统实现 Bluetooth HID Device Profile
- Windows 电脑具有蓝牙功能
- 源码构建需要 JDK 17 或更高版本、Android SDK Platform 34；当前验证环境使用 Temurin JDK 21

## 首次使用

1. 安装并打开应用，允许“附近设备”连接权限。
2. 进入“设备”，点按“进入配对模式”。Android 12 及以上仅在此时请求允许发现所需的 BLUETOOTH_ADVERTISE。
3. 在 Windows 打开“设置 → 蓝牙和设备 → 添加设备”，选择手机并完成配对。
4. 回到应用点按“刷新设备”，在已配对设备列表中连接电脑。
5. 返回“控制”，让目标 Windows 软件保持前台后开始操作。

应用只读取系统已配对设备，不执行蓝牙扫描；Manifest 不包含 BLUETOOTH_SCAN、定位、网络或存储权限。

## 页面与导航

底部四个标签固定为：

1. 控制
2. 设备
3. 自定义
4. 设置

页面只能通过点按标签、顶部设备状态或模式入口切换，不支持左右滑动翻页。各页使用可保存状态容器保留滚动与临时 UI 状态。竖屏使用浮动底栏；宽屏横屏使用右侧浮动导航栏。顶部状态卡、内容和导航均适配状态栏与手势导航安全区。

## 默认视频模式

首次启动创建两个受保护的系统模式：“默认视频模式”和“触控板模式”。旧配置会幂等迁移到 Schema v2，保留用户已有模式，并补齐缺失的系统模式与快捷控件。

| 操作 | 默认 HID 动作 |
| --- | --- |
| 上滑 | 键盘方向上，可用于上一条 |
| 下滑 | 键盘方向下，可用于下一条 |
| 左滑 / 右滑 | 键盘方向左 / 右 |
| 轻点 | 键盘空格 |
| 长按 | 键盘方向右 |
| 上一个视频 / 暂停视频 / 下一个视频 | Consumer Control 上一首 / 播放暂停 / 下一首 |
| 音量减小 / 静音 / 音量增加 | Consumer Control 音量动作 |

系统模式可以编辑、复制和恢复初始配置，但不能删除。用户可创建空白模式、复制任意模式、重命名、更换图标、排序、设置启动模式、导入/导出 JSON。音乐软件的上一首、下一首和播放/暂停可在动作选择器中绑定为标准 Consumer Control 动作。

## 控制区

每个控制模式可选择：

- 手势区：识别上下左右滑、轻点、双击、长按、圆圈、X、Z、勾和用户录制手势。
- 触控板：单指移动、轻点左键、双击、长按拖动、双指滚动和双指右键。
- 混合模式：在同一区域仲裁手势与触控板意图；需要同时在设置中启用混合模式。

默认控制页把主手势区、三枚视频快捷键和三枚音量键压缩在一个竖屏视口内；360 × 720 dp 模拟设备也无需滚动。上一个/下一个使用标准媒体上一首/下一首，因此同时兼容常见视频与音乐软件。音量减小和增加支持长按：500 ms 后开始，约每 100 ms 重复一次，最长 30 秒；静音为单次点按。

“触控板模式”使用独立、不可滚动的操作页。顶部“按键 开/关”可显示或隐藏左键、右键；隐藏后触控区自动扩展。设置中的“自定义手势快捷键”可分别绑定双指、三指及可选四指动作。

## 配置与动作模型

持久化配置使用 Schema v2，核心结构为 AppConfig、ControlProfile、ControlItem、GestureBinding、ProfileAppearance、GestureTemplate 和 TrackpadSettings。配置由 StateFlow 提供单一进程内状态源，并通过 SharedPreferences 保存 JSON 快照。

可配置动作包括：

- USB HID 键盘键
- 含修饰键的快捷键
- Consumer Control 媒体键
- 鼠标左/右/中键、按下、释放和相对移动
- 水平/垂直滚动；兼容路径使用 Shift + 滚轮表达水平滚动、Ctrl + 滚轮表达缩放
- 应用内动作
- 按下、释放、等待、重复组成的高级序列

当前 HID 描述符支持播放/暂停、上一首、下一首、停止、快进、音量增加、音量减小和静音八个 Consumer usage。组合滚动会在发送滚轮前按下修饰键，并在成功、异常或取消路径中保证释放。单个高级序列最多 64 步、静态总延时最多 30 秒，并校验按下/释放平衡。

配置 JSON 最大 2,000,000 字符。导入会校验 Schema、类型、ID、引用、数量、动作和手势边界；重复或保留的模式 ID 会安全重建，不覆盖系统模式。损坏的本地配置会备份截断副本并恢复默认配置。

## 自定义手势

自定义手势编辑流程为：

1. 录制同一单笔轨迹 3 次。
2. 检查与内置手势及已有自定义手势的相似冲突。
3. 再绘制 5 次验证识别稳定性。
4. 选择动作；存在冲突时必须明确允许覆盖。
5. 保存手势，再保存控制模式后生效。

手势识别在后台调度器运行，结果在控制区显示名称、置信度和成功/失败反馈。

## 输入安全与流畅性

- PointerActionQueue 使用单一工作协程、有限队列和相邻移动/滚动合并，避免每个采样点启动独立协程；离散按键顺序保持不变，释放事件预留容量。
- 触控板只有一个 MotionEvent 入口，按 pointerId 跟踪指针；状态机明确仲裁单指移动、拖动、双指滚动/缩放/右键、三指快捷动作和默认关闭的四指扩展。
- 自适应压力引擎融合 pressure、触点尺寸、接触面积与时长；支持中值窗口、EMA、异常值钳制、设备可靠性判断、校准与无可靠压力传感器时的模拟回退。
- OrderedLongPressDispatcher 串行化长按开始、重复和结束，避免 UP 越过尚未完成的 DOWN。
- 页面切换、模式切换、断开连接、进入后台、会话替换和执行器关闭都会取消在途动作并走统一释放路径。
- 持有键或鼠标按钮超过 30 秒会触发 watchdog 释放。
- 高级序列失败、取消、超时或运行结束仍有未配对按下状态时，会尽力释放已按下输入。

这些措施降低输入堆积和“卡键”风险，但真实蓝牙传输仍受手机厂商实现、Windows 蓝牙栈和无线环境影响。

## 已知限制

- Android 模拟器不能充当真实 Bluetooth HID Device。本次没有完成真实 Android 手机到 Windows 的首次配对、冷启动重连、方向键、鼠标、滚轮和媒体键端到端验收。
- 纯 HID 只能发送输入，不能读取 Windows 当前播放曲目、音量、静音状态、窗口内容或动作执行结果。
- 键盘、方向键、空格和滚轮作用于 Windows 当前活动窗口；不同软件的快捷键语义可能不同。
- Bluetooth HID Device Profile 是否可用取决于手机厂商系统；应用层无法补齐缺失的系统 Profile。
- 当前交付 APK 为 Android 开发签名，不是商店生产签名。
- Windows 应用对 Shift + 滚轮水平滚动、Ctrl + 滚轮缩放的解释不同，需在目标软件逐项验收。
- 当前版本保持标准键盘/鼠标/Consumer HID，不实现 Windows Precision Touchpad 描述符；后者建议作为独立实验分支。
